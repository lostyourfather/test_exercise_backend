# Тестовое задание

Состоит из следующих этапов:
  - Написание `docker-compose.yml` скрипта для разворачивания 
    - базы данных postgres, с указанием: названия БД, пользователя и пароля, 
    - основного приложения с зависимостями.
  - Написать модели для заданной структуры базы данных и с помощью `alembic` 
    сгенерировать и провести миграции в БД
  - Написать несколько запросов на запись и чтение данных
  - Дополнительно можно написать CRUD API: Create / Read / Update / Delete на каждую запись (авторизация не нужна)

## Структура базы данных

таблица `user`:
  - **id** int
  - **name** int
  - **books** книги

таблица `category`:
  - **id** int
  - **name** str

таблица `tag`:
  - **id** int
  - **name** str

таблица `book`:
  - **id** int
  - **name** str
  - **category_id** категория книги
  - **author_id** автор
  - **tags** список тегов (many to many)

## Запросы

Необходимы запросы на создание каждой сущности из схемы раздельно 
и запрос на создание книги с её автором одной транзакцией.

Для демонстрации оставьте скрипт на заполнение базы: по 5-10 записей, не больше.

Также нужно создать запрос на загрузку списка книг, с автором, категорией и 
списком тегов к каждой книге. Постарайтесь решить проблему N+1 запросов.
